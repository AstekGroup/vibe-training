Durée : 15 minutes

  • Algorithme Monte-Carlo pour estimer π

  • Affichage de la convergence vers π
  • Technologies : Python + Matplotlib ou JavaScript + Canvas

💡 Approche : Explication algorithme + visualisation

📋 Consignes

  • Choisissez 1 exercice parmi les 3 proposés (selon vos préférences)

  • Prompts libres : Pas de template imposé, expérimentez naturellement
  • Itérez rapidement : Si le résultat ne convient pas, reformulez et relancez

🛡️ Nouvelles menaces 2025

🎯 Agent Hijacking

  • Définition : Détournement d'agents autonomes
  • Cas documenté : Claude Code pour extorsion automatisée
  • Source : Anthropic (Août 2025)
  • Défense : Sandbox, permissions limitées

🔴 Context Poisoning via MCP

  • CVE-2025-49596 : RCE critique (CVSS 9.4)
  • CVE-2025-6514 : Autre vulnérabilité critique (CVSS 9.6)
  • Attaques : Tool Poisoning, Rug Pull
  • Défense : Validation outils, audit trails

🛡️ Tool Abuse et Data Exfiltration

⚔️ Tool Abuse - Cas Nx (Août 2025)

  • Premier cas : Malware exploitant CLI d'assistants IA
  • Données volées : Clés SSH, tokens npm, .gitconfig
  • Méthode : Hijacking Claude, Gemini, q CLI
  • Source : Snyk (Août 2025)

💾 Data Exfiltration avancée

  • Triple encodage Base64 : Dissimulation données
  • Repos GitHub publics : Exposition indirecte
  • Timing attacks : Information via délais
  • Proxy malveillants : Interception APIs

📋 Rules Engineering

Maîtriser les fichiers de contexte persistant

🎯 Sous-section intégrée au Contrat de Contexte (9h00-10h00)

Les fichiers de règles (CLAUDE.md, .cursorrules, etc.) sont la persistance pratique du contrat de contexte.

📝 Qu'est-ce qu'un fichier de règles ?

Fichier de configuration persistant définissant instructions, comportements et contraintes pour l'agent IA

🔑 Formats principaux

  • CLAUDE.md (Claude Code)
  • .cursorrules (Cursor)
  • .continuerules (Continue)
  • .aicontext (générique)

⚡ Exercice pratique

Audit + Création + Test de règles modulaires pour votre projet

🔍 Audit de règles existantes

✅ Fichier 1 - Bonnes pratiques

Tu es un développeur Python expert. Utilise toujours des type hints. Écris des docstrings pour toutes les fonctions. Privilégie la lisibilité à la performance.

❌ Fichier 2 - Règles problématiques

Fais du bon code. Sois efficace. N'oublie pas les tests. Attention à la sécurité.

🔬 Questions d'analyse

  • "Lesquelles donnent des instructions actionables ?"
  • "Comment mesurer l'efficacité de ces règles ?"

🤖 Qu'est-ce que Google ADK ?

📚 Définition

Google Agent Development Kit (ADK) est un framework de développement d'agents IA permettant de créer des agents conversationnels personnalisés avec Gemini.

🔑 Concepts Clés ADK

1. Agents ADK

Définition : Entité IA autonome avec instructions, outils et mémoire

  • Instructions système (personnalité, comportement)
  • Outils disponibles (functions, API calls)
  • Gestion de la mémoire (conversation, contexte)
  • Configuration du modèle Gemini

2. Tools (Outils)

Capacités :

  • Function Calling : Appel fonctions Python depuis l'agent
  • API Integration : Connexion services externes
  • Custom Tools : Outils métier spécifiques

3. Memory & Context

Gestion :

  • Session Memory : Conversation temporaire
  • Persistent Memory : Stockage long terme (DB)
  • Context Management : Gestion optimale contexte

# agent/tools.py
from typing import Dict
import ast

def analyze_code_complexity(code: str, language: str = "python") -> Dict:
    """Analyse la complexité d'un code source."""
    try:
        if language.lower() == "python":
            tree = ast.parse(code)

            # Comptage des éléments
            functions = [n for n in ast.walk(tree) if isinstance(n, ast.FunctionDef)]
            classes = [n for n in ast.walk(tree) if isinstance(n, ast.ClassDef)]

            # Calcul de métriques
            lines_of_code = len(code.split('\n'))
            num_functions = len(functions)
            num_classes = len(classes)

            # Score de complexité
            complexity_score = (num_functions * 2 + num_classes * 3) / max(lines_of_code / 10, 1)

            return {
                "lines_of_code": lines_of_code,
                "num_functions": num_functions,
                "num_classes": num_classes,
                "complexity_score": round(complexity_score, 2),
                "assessment": _get_complexity_assessment(complexity_score)
            }
    except Exception as e:
        return {"error": f"Erreur d'analyse: {str(e)}"}

def _get_complexity_assessment(score: float) -> str:
    if score < 1.0:
        return "Simple - Bonne lisibilité"
    elif score < 2.0:
        return "Modéré - Complexité acceptable"
    elif score < 3.0:
        return "Élevé - Envisager refactoring"
    else:
        return "Très élevé - Refactoring recommandé"

📝 Code Exemple : check_code_style

# agent/tools.py (suite)
def check_code_style(code: str, language: str = "python") -> Dict:
    """Vérifie le respect des conventions de style."""
    issues = []

    if language.lower() == "python":
        lines = code.split('\n')

        for i, line in enumerate(lines, 1):
            # Vérification longueur de ligne (PEP8 : max 79 caractères)
            if len(line) > 79:
                issues.append(f"Ligne {i}: Dépasse 79 caractères")

            # Vérification espaces autour opérateurs
            if '=' in line and not ('==' in line or '!=' in line or '<=' in line or '>=' in line):
                if '= ' not in line and ' =' not in line:
                    issues.append(f"Ligne {i}: Manque d'espaces autour de '='")

        return {
            "total_issues": len(issues),
            "issues": issues[:10],  # Limite aux 10 premiers
            "status": "✅ Conforme" if len(issues) == 0 else f"⚠️ {len(issues)} problèmes"
        }

    return {"error": "Langage non supporté"}

# agent/config.py
from google.adk import Agent
from agent.prompts import VIBE_CODING_INSTRUCTIONS
from agent.tools import analyze_code_complexity, check_code_style
import os

def create_code_review_agent():
    """Crée un agent avec outils de code review"""

    agent = Agent(
        name="CodeReviewAgent",
        model="gemini-2.0-flash-exp",
        instructions=VIBE_CODING_INSTRUCTIONS + """

        Outils disponibles :
        - analyze_code_complexity: Analyse la complexité du code
        - check_code_style: Vérifie les conventions de style PEP8

        Utilise ces outils pour fournir des reviews complètes et constructives.
        """,
        tools=[analyze_code_complexity, check_code_style],
        temperature=0.3,  # Plus déterministe pour code review
        api_key=os.getenv('GOOGLE_API_KEY')
    )

    return agent

🎯 Points Clés

  • Les outils sont de simples fonctions Python avec type hints
  • Les docstrings sont utilisées par l'agent pour comprendre l'outil
  • Temperature basse (0.3) pour résultats plus déterministes en code review
  • Les outils sont passés en liste au paramètre tools=[]
