🔬 Anatomie technique d'un agent

Les 4 composants essentiels

1. Planning loop : Décomposition de tâches complexes Cycle agent typique : 1. Receive task → 2. Break down → 3. Execute step 4. Evaluate → 5. Adjust plan → 6. Continue/Complete

🎯 Exemple concret

Tâche : "Créer une API de gestion d'utilisateurs"

→ Décomposition : Modèles, Routes, Sécurité, Tests, Documentation

🛠️ Tools & Memory Management

2. 🛠️ Tool use

  • File system : Read, write, search
  • Version control : Git operations
  • Web : API calls, documentation
  • Execution : Run code, tests, build

3. 🧠 Memory management

  • Context window : Conversation immédiate
  • Session memory : État projet en cours
  • Long-term memory : RAG
  • Shared memory : Entre agents

4. 🔄 Self-correction

  • Error detection : Parsing erreurs compilation
  • Retry strategies : Tentatives avec ajustements
  • Fallback mechanisms : Solutions alternatives

📋 Maîtriser les fichiers de contexte persistant

🎯 Objectif : Exercice progressif (75min)

  1. Étape 1 (20min) : Audit de règles existantes
  2. Étape 2 (30min) : Création de règles modulaires
  3. Étape 3 (25min) : Test en conditions réelles

🔍 Audit de règles existantes

✅ Fichier 1 - Bonnes pratiques

Tu es un développeur Python expert. Utilise toujours des type hints. Écris des docstrings pour toutes les fonctions. Privilégie la lisibilité à la performance.

❌ Fichier 2 - Règles problématiques

Fais du bon code. Sois efficace. N'oublie pas les tests. Attention à la sécurité.

🔬 Questions d'analyse

  • "Lesquelles donnent des instructions actionables ?"
  • "Comment mesurer l'efficacité de ces règles ?"

⭐ Exemple de règles avancées

# Contexte : API fintech haute performance Tu es un senior backend engineer spécialisé en systèmes financiers. STACK TECHNOLOGIQUE : - Python 3.11+ avec FastAPI - PostgreSQL avec SQLAlchemy async - Redis pour caching - Pytest pour tests CONTRAINTES RÉGLEMENTAIRES : - PCI DSS Level 1 compliance - SOX audit requirements - GDPR pour données EU STANDARDS QUALITÉ : - Type hints obligatoires (mypy strict) - Couverture tests > 95% - Logging structuré (JSON) - Métriques OpenTelemetry SÉCURITÉ : - Jamais de secrets en dur - Validation inputs avec Pydantic - Audit trail pour toutes mutations - Rate limiting sur tous endpoints

🏗️ Création de règles modulaires

🛠️ Exercice pratique (binômes) - 30min

Étape 1 : Règles générales (10min)

Créez rules-general.md avec :

  • Principes qualité universels
  • Standards sécurité de base
  • Patterns communication préférés

Étape 2 : Règles spécifiques (15min)

Créez 3 fichiers spécialisés :

  • rules-backend.md : API, databases, performance
  • rules-frontend.md : UI, UX, responsiveness
  • rules-mobile.md : Cross-platform, performance

🎯 Points clés

  • L'itération : "Rules parfaites n'existent pas"
  • L'importance du test : "Rules non testées = inutiles"
